MySQL学习（二）

发表于 2021-11-12 分类于 数据库 本文字数： 425
一条SQL更新语句是如何执行的.

更新语句流程（与查询操作一样的一些步骤）：
连接器、分析器、优化器、执行器（找到行数据，更新数据）。

不同的是：日志模块。

redo log（引擎层，InnoDB引擎）

WAL技术（write ahead logging），先写日志，再写磁盘。每一次更新操作都需要写进磁盘，IO成本、查找成本很高。每一次更新，先把记录写到redo log里，并更新内存，在空闲的时候，将此操作更新到磁盘。或者log空间满了，进行更新操作。（文件固定大小）

  • write pos
  • checkpoint

两阶段提交

  1. redo log prepare
  2. binlog
  3. redo log commit

当在2操作崩溃时，重启，redo log处于prepare，binlog失败，回滚。
当在3操作崩溃时，重启，再次提交。

binlog（server层）

逻辑日志，记录的是这个语句的原始逻辑。
追加写入，binlog文件写到一定大小后，会切换到下一个，并不会覆盖以前的日志。
