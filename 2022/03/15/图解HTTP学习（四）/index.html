Session, Cookie, Token

Session, Cookie

HTTP 无状态协议。
Cookie 跟踪用户会话。
Session机制，基于唯一ID识别用户身份的机制。采用Cookie形式实现。

服务器把所有用户的Session都存储在内存中。放入session的对象要小。


服务器识别Session的关键是依靠一个名为JSESSIONID的Cookie。首次服务器向浏览器分配一个唯一的Session ID，并通过名为JSESSIONID的Cookie发送到浏览器，浏览器在后续访问时总是附带此Cookie，服务器可以识别用户身份（状态）。
也可以设置一个Cookie，如：记住用户选择的语言。

Token

生成token，验证token。不保存token。
给客户端发一个令牌token，里面包含用户的user id，同时对数据做一个签名，比如用HMAC-SHA256算法，加上一个私有密钥，对数据做一个签名，把数据和签名一起作为token。
不保存token，只用算法和密钥重新计算签名，进行对比。

区别

session会话，利用cookie的方式实现状态保存。客户端需要保存cookie，每次请求都需要带cookie。同时服务端需要保存客户信息和session id，以方便对比记录状态。内存开销增大。
token通过签名的方式，实现不保存也可以记录状态。
