从头搭建一个网站

发表于 2022-07-04 分类于 Java 本文字数： 1.6k
基于SpringBoot从头搭建一个网站.

环境

  • 开发工具：IDEA
  • 网站系统框架：SpringBoot
  • 依赖包安装管理：Maven安装
  • 数据库：Oracle安装
  • 数据库ORM：mybatis, hibernate
  • 数据库连接池：Druid(阿里)
  • 前端模板：thymeleaf

采用SpringBoot开发，数据库使用连接池和ORM（建立实体类和数据库表之间的关系，从而达到操作实体类就相当于操作数据库表的目的）

结构设计

  • entity：建立实体类和数据表之间的映射，实现持久化
  • dao：实现对实体类的增删查改
  • service：调用dao层接口，实现一些功能，以供controller调用
  • controller：定义HTTP接口，接收请求，并调用相关Service完成操作，返回响应结果

进阶：

  • constants：常量包，比如Errocode
  • util：工具类
  • intercepter：拦截器，在进入页面前都要安排验证身份？
  • exception：统一定义的异常

实现

数据库设计

定义各表：表名称，主键，列，列属性，列约束，外键。

编写SQL语句

注意：由于Oracle不能在建表的时候声明自增主键，所以需要新建序列和触发器，实现自增，同时在代码层面，也需要使用序列，此时需要禁用触发器，否则每次会自增+2.
同时需要注意外键。

entity层

实体类与数据库表的映射，实现数据持久化。
lombok里的@Data可以自动实现setter和getter以及toString()，不需要自己再声明实现。以及无参构造函数和全参构造函数的实现。

dao层

定义接口，定义对数据库的增删查改的函数，即对实体类的操作。
继承自JPA，大部分已实现。对于特殊操作，需要自己手写SQL语句。如update.

Service层

根据需求分析，将具体业务转成具体代码，主要调用dao层的一些增删查改操作。
有时会使用到事务，使用注解@Transactional.

controller层

页面和后端的交互主要由controller实现，@PostMapping和@GetMapping注解，实现对用户的请求的获取。定义对应URL请求的controller，进行业务处理。

前端

模板引擎thymeleaf可实现。通过th标签+元素名在前端和后端之间传输数据。
