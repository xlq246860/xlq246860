SQL语法基础

发表于 2022-07-16 分类于 数据库 本文字数： 2.7k
SQL语句如何写.

distinct

返回唯一不同的值。
例：每个名字仅被列出一次。
select distinct name from students;

order by

默认的排序规则是ASC：升序，即从小到大。DESC：倒序。

limit

分页查询。从结果集中截取出第M~N条记录，通过limit <n-m> offset <m>实现，表示从m号记录开始，最多取(n-m)条，也就是取第m~n条记录。
页面数量：pageSize=n-m
页索引：pageIndex=m
MySQL中简写为：LIMIT 15 OFFSET 30; -> LIMIT 30, 15;
例：把结果集分页，每页三条记录

聚合查询

统计总数、平均数这类计算，使用聚合函数进行查询。

sum/avg/count

  • count(*)表示查询符合条件的所有列的行数，查询结果是二维表，只有一行一列。
    例：统计有多少男生
  • sum 计算某一列的合计值，该列必须为数值类型
  • avg 计算某一列的平均值，该列必须为数值类型
    例：计算男生的平均成绩
  • max 计算某一列的最大值
  • min 计算某一列的最小值

group by

分组聚合。

  • 例：统计各班人数情况
  • 例：统计各班的男女生人数和平均分
    使用多个列进行分组。
  • 错误例子： 上述例子会报错，因为在任意一个分组里，只有class相同，name不同，不能把多个name放入一行记录中。

使用group by时，select子句只能存在以下三种元素：

  1. 常数
  2. 聚合函数
  3. group by子句中指定的列名

having

where关键字无法与聚合函数一起使用。

  • 例：找出平均分大于80的班级

多表查询

笛卡尔乘积

select * from students, classes;
结果集 中 行数为两表的行数之积，列数为两表列数之和。
尽量不使用。

连接查询

连接查询对多个表进行JOIN运算，先确定一个主表作为结果集，然后把其他表的行有选择地连接在主表结果集上。

  • INNER JOIN：两表都存在的记录
    确定主表，使用from <表1>的语法，再确定需要连接的表inner join <表2>。然后确定连接条件，on <条件>，s.class_id=c.id表示student表的class_id与classes表的id相同的行需要连接。
  • LEFT OUTER JOIN：左表存在的记录
  • RIGHT OUTER JOIN：右表存在的记录
    右表有些行的id在左表中并不存在，所以会将左表列中值填为null。
  • FULL OUTER JOIN：左右表都存在的记录
    相同id的连接，不存在的新起一行，不存在的填充为Null.
